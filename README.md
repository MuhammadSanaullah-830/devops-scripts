---

## 🛡 Best Practices

- Never hardcode secrets in scripts
- Use environment variables for credentials
- Follow least privilege principle (AWS/IAM)
- Test scripts before production use
- Keep scripts modular and reusable
- Add proper comments in scripts

---

## ⚠️ Common Mistakes

- Running scripts without testing
- Storing passwords or keys in code
- Mixing unrelated scripts in same folder
- No documentation for scripts
- Running scripts as root without need

---

## 🔍 Troubleshooting

If a script fails:

```bash
chmod +x script.sh
```

- Check file permissions
- Validate dependencies installed
- Check logs/output errors
- Run script step by step
